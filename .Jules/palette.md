## 2024-05-18 - WebXR Accessible Status Overlay
**Learning:** 3D text meshes in WebXR (Three.js) are completely inaccessible to screen readers natively. Status updates (like object grab states, scaling feedback, or loading state) conveyed via in-world debug panels fail to communicate critical state to low-vision users using assistive technologies.
**Action:** When adding or managing WebXR status feedback, utilize WebXR's `dom-overlay` feature to mirror 3D status strings to a hidden or non-obstructive 2D DOM element with `role="status"` and `aria-live="polite"`. Ensure the DOM overlay container passes pointer events to the canvas (`pointer-events: none`) while interactive parts keep `pointer-events: auto`. Position 2D elements sufficiently above the bottom (e.g. `bottom: 80px`) to prevent visual obstruction of the persistent `ARButton`.
