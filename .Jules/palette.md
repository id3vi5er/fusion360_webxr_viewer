## 2024-05-24 - WebXR Status Messages Accessibility
**Learning:** WebXR 3D text alone is completely invisible to screen readers. Creating an accessible WebXR experience requires explicitly requesting the `dom-overlay` feature in the XR session to project a 2D HTML layer over the 3D canvas. Furthermore, this 2D layer needs `pointer-events: none` to avoid blocking 3D scene interactions, while the individual interactive or readable elements (like buttons or toasts) must reinstate `pointer-events: auto`. Finally, dynamically updating text requires an `aria-live` region (`role="status" aria-live="polite"`) for screen readers to announce state changes (e.g., "Ready", "Grabbed") without the user needing to refocus.
**Action:** Always wrap WebXR experiences with an accessible DOM overlay containing `aria-live` regions for critical state changes, and carefully manage `pointer-events` on the overlay stack.
