## 2024-05-24 - WebXR Accessible Toast Notifications
**Learning:** In immersive WebXR sessions, relying solely on 3D objects (like floating text planes) for status updates is not accessible. Traditional 2D DOM elements are completely hidden when an AR session starts unless the `dom-overlay` feature is explicitly requested and configured. Even then, an `aria-live` region is necessary for screen readers to announce dynamic updates in the AR environment.
**Action:** Always request the `dom-overlay` feature in WebXR sessions when UI feedback is needed, and sync 3D status text to a visually hidden or unobtrusive 2D `aria-live="polite"` element within the overlay root. Ensure the overlay container has `pointer-events: none` while interactive children have `pointer-events: auto` to prevent the UI from swallowing canvas interactions.
