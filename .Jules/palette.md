## 2024-05-24 - WebXR DOM Overlay Accessibility
**Learning:** In WebXR (Three.js), simply rendering text onto a 3D canvas (via `CanvasTexture`) makes the UI completely invisible to screen readers and difficult to interact with. However, the `dom-overlay` WebXR feature allows rendering standard, accessible 2D HTML DOM elements over the AR session.
**Action:** When creating status updates or UI in WebXR applications, use `dom-overlay` and render standard HTML elements with appropriate ARIA roles (e.g., `role="status"`, `aria-live="polite"`) instead of relying solely on 3D meshes for text. Crucially, the root overlay container must have `pointer-events: none` to pass hits to the WebXR canvas, while interactive children (like toasts) must explicitly set `pointer-events: auto`.
