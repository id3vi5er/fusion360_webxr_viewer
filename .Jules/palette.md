## 2023-10-27 - WebXR DOM Overlays and Accessibility

**Learning:** WebXR overlays (using `dom-overlay` required feature in Three.js ARButton) are highly sensitive to pointer events. To create an accessible overlay notification (like a toast), you must structure a wrapper element that spans the whole canvas but has `pointer-events: none` to pass clicks down to the Three.js hit-test mechanisms. However, for specific interactive elements or toasts within that overlay, you must set `pointer-events: auto` to ensure they are readable and potentially interactive, whilst retaining necessary ARIA attributes like `role="status"` and `aria-live="polite"`. The overlay must also not overlap with the generated Three.js `ARButton` at the bottom of the screen (e.g., placing toasts at `bottom: 80px`).
**Action:** Always structure WebXR overlays with a full-screen, pointer-transparent root container and pointer-opaque interactive child elements. Position bottom-aligned HUD elements above 80px to avoid ARButton overlap.
