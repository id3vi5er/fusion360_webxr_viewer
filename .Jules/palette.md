## 2024-05-14 - WebXR Status Updates Accessibility
**Learning:** 3D text meshes within WebXR scenes are entirely opaque to screen readers, making critical status updates inaccessible to visually impaired users navigating the application. To provide accessible real-time feedback in AR/VR, developers must pair WebGL rendering with a 2D HTML element. This element needs to reside within a DOM overlay configured in the WebXR session.
**Action:** When implementing status messages or live text updates in WebXR/Three.js projects, always instantiate a corresponding 2D HTML container (e.g., a `div`) with `role="status"` and `aria-live="polite"`. Mount this container within a `dom-overlay` specified in the `ARButton`'s `optionalFeatures` to ensure screen readers can read the content while remaining visible in passthrough.
