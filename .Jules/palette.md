## 2024-05-24 - Accessible Notifications in WebXR Canvas
**Learning:** 3D text meshes (like the debug mesh used in `index.html`) are inherently inaccessible to screen readers because they exist within the WebGL canvas. When building WebXR applications, standard visual status updates or debug text will not be announced to users relying on assistive technology.
**Action:** When creating accessible status updates or notifications in WebXR, always implement a 2D HTML element with appropriate ARIA attributes (e.g., `role="status"`, `aria-live="polite"`) nested inside a `dom-overlay` container. This container must be configured via the `ARButton`'s `optionalFeatures` and `domOverlay` options. This allows screen readers to access and announce dynamic updates while in the AR/VR session.
